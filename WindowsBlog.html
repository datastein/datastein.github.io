Building a Data Science Blog on the Window's platform using Anaconda, Pelican, and Github

Posted on Sun 02 June 2019 in posts

There are many great reasons that a Data Science blog might be conducive to your career. The first reason is that it allows potential employers or collaborators a chance to see that you possess a strong aptitude for unique problem solving that often canâ€™t be achieved during a relatively short interview. It also allows one to see whether you possess an ability to clearly articulate your data interpretation and statistical methods effectively. In a world where your statistics can be on-point and you can use the best methods and yet still have the wrong answer, it is important that you clearly lay out your hypotheses and assumptions to your readers and reviewers.

That said, we need a means of getting said blog up-and-running. When I started this blog, I wanted the fastest way to create beautiful content with a minimum of trouble. I also wanted an easy way to demonstrate what my code looks like to an outside reader. Thus, I turned to the use of a static site generator. While there are few different generators that I could have used, I have been spending a lot of time on GitHub and Jupyter recently. Therefore, I have chosen to use Pelican which is written in Python. This is a great tool that makes if very easy to generate a beautiful looking blog.

While there are several really great blogs out there that cover how to generate a static website on Pelican, there are relatively few that cover building a static website in the Windows environment with Anaconda Python distribution.

Requirements¶

There are several minimum requirements to getting your blog running. This section will run through how to install each components. The bare minimum requirements are:

  1. Install Python 2.7.x, Python 3, or Python 3.5 from Python
  2. Set up a virtual environment
  3. Install Pelican Package

1. Install Python ¶

Assuming you are starting at square one, you might first need to install Python. I recommend using Anaconda Python Distribution. There are two ways to start on Windows:

  1. Get the Anaconda Navigator ap from the appstore and follow the installation instructions.
  2. Download the Anaconda Installer directly. Follow the installation instructions for Windowns available on the Windows download page.

Once you have things downloaded, check that things are working right by using command prompt language. 'Right click' on the Start menu and select Command Prompt or Anaconda Prompt. Start the Python interpreter by typing: **> Python**</span>.

You should see a message that matches the version of Anaconda installed:

c:\Users\jstein=python
Python 3.5.2 [Anaconda 4.1.1 (64-bit)\ (default, Jul 5 2016, 11:41:13)

To leave the Python shell, press Ctrl+D or type the command > exit () </span>

To verify that Anaconda has installed properly, from the Windows command shell type:

conda info --all

It should print out a notification on the version and type of Anaconda installed.

2. Set up a virtual environment ¶

Before we get started making changes to our new tools, we should set up a virtual environment using VirtualEnv. A virtual enviroment is an isolated python environement where you can protect your source code from Python library/package dependancy conflicts. This environment contains a fresh copy of the Python binary with its own copy of the Python standard library. Read more about it here.

For this step, let's create a virtual environment called pelicanTemp. Open up the Anaconda terminal from the Start menu and type the following:

conda create -n pelicanTemp python=2

Once this environment is created, you can enter and leave the PelicanTemp environment again at will by typing, respectively:

conda activate pelicanTemp

or

conda deactivate

While you are in the pelicanTemp environment, you should see the following text at the command prompt:

(pelicanTemp) C:\Users\mj_st>

3. Install Pelican ¶

We are now ready to install the Pelican package. From inside your Pelican virtual environment,execute:

pip install pelican markdown

Once Pelican is installed, continue the blog set-up by doing the following:

  • Create a local folder for your blog. This is where we will add our blog content and styles. We'll refer to this as 'DS_blog' but feel free to name this whatever you want. My folder was saved in 'C:\Users\UserName\Documents\Python_Scripts\DS_Blog'.

  • Change the directory in Anaconda to your blog's folder:
    • cd C:\Users\UserName\Documents\Python_Scripts\DS_Blog


  • Create a file in your blog folder called '.gitignore' and add the content from this file

  • Create a file called 'requirements.txt'</mark> in WordPad (or similar text editor). We will use this file to store a list of packages that will make developing and publishing a Pelican blog site easier. From your text editor, add:
beautifulsoup4~=4.3.2
feedgenerator~=1.7
html5lib~=0.999
Markdown~=2.5.2
Pygments~=2.0.2
pyreadline~=2.0
smartypants~=1.8.6
typogrify~=2.0.7
gitpython


** !!! Warning !!! **
If you have any other platform **BUT** Windows you can add in a line such as `ghp-import==1.5.1` to your 'requirements.txt' folder.

'ghp-import' however has an [error](http://docs.getpelican.com/en/3.6.3/tips.html) that makes it impossible to use on Windows currently and it needs its own seperate installation code. Run the following AFTER you have installed the requirements folder above:
`pip install https://github.com/chevah/ghp-import/archive/win-support.zip`

Save the 'requirements.txt' file in your blog folder. From the Anaconda shell, run:

pip install -r requirements.txt

Create your Data Science Blog ¶

Once you have the basic set-up online, you should be all set to create your first blog! To do this type pelican-quickstart when you are in DS_blog folder. This will activate the pelican interface. For the majority of the answers, go ahead and just hit Enter and accept the default value. The ones that you don't want to accept as default are:

  1. Title of the Website
  2. The author of the website,
  3. n for URL prefix
  4. Timezone

After running the pelican-quickstart function, you should have two new folders in your DS_Blog folder: 'Content' and 'Output', along with a few extra files: 'pelicanconf.py' and 'publishconfig.py'.¶

In order to activate the plugin we need to modify the 'pelicanconf.py' folder and add these lines at the bottom:

MARKUP = ('md', 'ipynb')

PLUGIN_PATH = './plugins'
PLUGINS = ['ipynb.markup']

I like to use a program like Visual Studio to do these edits but choose the program of your choice. These lines will direct Pelican to activate the generated plugin file.

Get a Github Account and Complete File Setup ¶

Go to GitHub and create an account. GitHub Pages is a free Github feature that will allow you to deploy a static site. To enable this, do the following:

  1. Sign up for a Github account as shown above.
  2. Follow the directions to create a repository called username.github.io where username is your GitHub username.
  3. From the Anaconda Navigator app, open up a Jupyter Notebook and create some basic content in Markdown. Save the file as a .ipynb file in the content folder of your blog.
  4. Create an nbdata file that has the following structures with your own specifics:


Title: Some Post
Slug: Some-post
Date: 2019-05-27 18:00
Category: posts
Tags: python blogs
author: Your Name
Summary: Wow! Cool. I have a blog

My 'Content' folder now contains two files 'Some_post.ipynb' and 'Some_post.nbdata'. You need similar files for every post that you create.

While most of the above descriptors are self-explanatory, here is a brief descriptor for a few of the above fields:

  • Slug: This is the path with which the post will be accessed on the server (i.e http://username.github.io/Some_post)
  • Tags: Seperate with a space, this list can be anything.
  • Summary: A brief description of your post.

If you want to add images to your file, you need to store them in a local spot. To do this set up a new folder in DS_Blog and name it 'img'. Then, save an image to that folder. Where you want the image in your page, you can now link to the file using the command:

`![Image Description](img/picturefile.png "beautiful pic")`

In order to activate the plugin we need to modify the 'pelicanconf.py' folder in your main blog folder and add these lines at the bottom:


pelican content
cd output
start "" "http://localhost:800/"

python -m pelican.server

Once this is entered in, open up a web browser and navigate to 'http://localhost:800/' You should now see the default skeleton and template for the first blog post.

Commit your Files to GitHub ¶

In order to get your files loaded onto GitHub, you need to create a local git repo:

git Init

Next, you can connect to the repo that you created when we signed up for GitHub:

git remote add origin https://github.com/username/username.github.io.git
git remote -v

Go ahead and create a new branch now for the pelican source:

git checkout -b pelican

Now that we have a connection to GitHub, we need to modify the configuration of your Pelican blog so that the URLs navigate correctly:

  • Open up 'publishconfig.py'. in Visual Studio and edit the SITEURL to be equal to http://username.github.io.

After this you can use the standard commands to push and pull data onto GitHub. A very useful program is 'ghp-import' which allows one to copy a directory to the gh-pages branch of the repository without having to do individual commands.

** !!! Warning !!! **
If you get an error saying `Permission Denied (publickey)` when you push to GitHub, then you have not made any SSH keys. Checkout this [site](https://gist.github.com/adamjohnson/5682757) for more instructions on this error.

Once you are set to publish, go ahead and publish your blog with the following commands:
git add .
git commit -a -m %1
git push -u origin pelican
pelican content -o output -s pelicanconf.py
ghp-import output -r origin -b master
git push origin master
git checkout pelican

That is it! Navigate to your URL and you should now see your content.
